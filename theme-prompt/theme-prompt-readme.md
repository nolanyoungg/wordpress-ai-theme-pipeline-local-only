# Theme Prompts (Ollama)

This folder is for storing reusable theme prompts you run locally with Ollama (for `scripts/run-local-ollama-workflow.ps1`).

The goal of every prompt in this folder is to produce a complete, production-quality classic WordPress theme plus a matching static GitHub Pages preview.

## Absolute Requirements (Non-Negotiable)

Your prompt must require the generated output to meet all of the following:

### 1. Outputs Created

- A new versioned theme folder under `wp-content/themes/nolan-young-showcase-theme-xNN/` (never overwrite older versions).
- A matching static preview under `docs/themes/nolan-young-showcase-theme-xNN/`.
- Update `docs/index.html` to add a gallery card linking to `themes/nolan-young-showcase-theme-xNN/index.html`.
- A ZIP under `zippedTheme/nolan-young-showcase-theme-xNN.zip` generated by running `scripts/validate-themes.ps1`.

### 2. No External Runtime Dependencies

- No cloud AI calls, no `OPENAI_API_KEY`, no paid APIs.
- No CDNs, no remote images, no remote fonts, no icon CDNs.
- Use local CSS/JS and inline SVG where needed.

### 3. WordPress Theme Quality (Classic Theme)

- Classic theme templates (PHP), not a block theme.
- Proper enqueueing in `functions.php`:
  - `wp_enqueue_style()` for `assets/css/theme.css`
  - `wp_enqueue_script()` for `assets/js/theme.js`
- Use safe escaping where applicable:
  - `esc_html()`, `esc_attr()`, `esc_url()`, `wp_kses_post()`
- Clean template structure and readable code.
- Meaningful CSS and JS (not tiny placeholder files).

### 4. Static Preview Quality

- `docs/themes/<theme-slug>/index.html` mirrors the WordPress front page layout and interactions as closely as possible.
- Uses only static HTML + CSS + vanilla JS.
- Uses relative paths:
  - `assets/css/preview.css`
  - `assets/js/preview.js`
- Includes accessible interactions (keyboard + focus states).
- Respects `prefers-reduced-motion` for animations.

### 5. "Full Blown Website" Expectations (Homepage)

Your prompt should require at least these sections on the front page and in the static preview:

- Hero: strong headline + subhead + primary CTA + secondary CTA
- Trust row: badges/logos/metrics (text-only ok)
- Services: 4-6 cards (WordPress builds, performance, maintenance, integrations, AI automation)
- Case studies / featured projects: at least 3 cards
- Process: at least 4 steps
- Testimonials: 2–3 entries
- Engagement models / pricing teaser: 3 options
- FAQ: 5+ accordion items with keyboard support
- Final CTA section (conversion-focused)

Require at least two interactive components:
- FAQ accordion
- One of: tabbed content, filterable grid, carousel, animated counters (must respect reduced-motion)

### 6. Must Be Distinct From Previous Version

If a previous version exists, require the new version to be materially different:
- New palette + typography scale + spacing rhythm
- Different hero layout and at least one different section layout pattern
- Do not reuse large chunks of the previous `theme.css` / `preview.css`

## Recommended Prompt Template

Copy this into a new file in this folder and replace the bracketed parts:

```text
TASK
Create the next versioned classic WordPress theme in this repo (nolan-young-showcase-theme-xNN), plus a matching static GitHub Pages preview.

HARD CONSTRAINTS
- Local-only AI (Ollama). No cloud AI, no CDNs, no remote assets.
- Classic WP theme with proper enqueueing and escaping.
- Static preview must mirror the WP front page and interactions.
- Update docs/index.html to add the new theme card.

EXPERIENCE + CONTENT
[Describe the site's business, positioning, and voice.]
Require: Hero, Trust, Services, Case Studies, Process, Testimonials, Engagement Models, FAQ accordion (5+), Final CTA.

VISUAL DIRECTION (MAKE IT CLEARLY DISTINCT)
[Define palette, typography direction, layout motifs, motion style, and what to avoid.]

INTERACTIONS
- FAQ accordion (keyboard + focus)
- [Second interactive component]
- Respect prefers-reduced-motion
```

## How To Run A Prompt File

Add your prompt as a plain text file in this folder (recommended extension: `.txt`). Commit it if you want the exact prompt used for a theme to be reproducible later.

From a fresh PowerShell window:

```powershell
cd C:\Users\Nolan\Documents\Codex\wordpress-ai-theme-pipeline-local-only

$env:OLLAMA_MODEL="qwen2.5-coder:32b"
$env:OLLAMA_HTTP_TIMEOUT_MINUTES="360"

powershell -ExecutionPolicy Bypass -File scripts/run-local-ollama-workflow.ps1 (Get-Content -Raw "theme-prompt\\YOUR_PROMPT_FILE.txt")

powershell -ExecutionPolicy Bypass -File scripts/validate-themes.ps1
git add -A
git commit -m "Add new theme version"
git push origin main
```
